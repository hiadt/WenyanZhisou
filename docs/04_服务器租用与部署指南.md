# 服务器租用与部署指南

## 1. 先说结论

如果你要认真跑比赛，不建议再用普通 16GB 内存电脑硬撑。普通电脑可以做演示，但正式跑本地 embedding/reranker，甚至本地 LLM，建议租 GPU 服务器。

推荐优先级：

1. 性价比正式版：RTX 4090 24GB，64GB RAM。
2. 更稳正式版：L20 48GB，64GB RAM 或 96GB RAM。
3. 预算充足版：A100 40GB/80GB，64GB RAM 以上。
4. 只调试：A10 24GB 或 RTX 3090/4090 24GB，32GB RAM 也能试。

## 2. 不同需求对应配置

### 轻量前端演示

```text
GPU: 不需要
CPU: 4-8 vCPU
RAM: 16-32GB
Disk: 30GB+
```

启动时使用：

```bash
pip install -r requirements-lite.txt
python web_demo.py --config config.online.json --no_llm --fallback_models --port 8091
```

### 正式小模型评测

```text
GPU: RTX 4090 24GB / A10 24GB / L20 48GB / A100
CPU: 8 vCPU+
RAM: 64GB 推荐
Disk: 100GB SSD+
Python: 3.10 或 3.11
CUDA: 选择平台预装 PyTorch + CUDA 镜像
```

适合加载 embedding 和 reranker。

### 本地部署 7B LLM

```text
GPU: L20 48GB / A100 40GB+ 更稳
CPU: 8-16 vCPU
RAM: 64GB+
Disk: 150GB SSD+
```

如果只是调用 DeepSeek、通义千问、硅基流动等 API，不需要本地部署 LLM，可以省掉这部分显存压力。

## 3. 服务器镜像选择

租服务器时优先选：

```text
Ubuntu 22.04
Python 3.10+
CUDA 12.x
PyTorch 2.x
```

如果平台提供“PyTorch + CUDA 已安装”的镜像，优先选这种，少踩 CUDA 安装坑。

## 4. 部署步骤

上传项目压缩包后：

```bash
unzip wenyan-zhisou-advanced.zip
cd wenyan-zhisou-advanced/competition
python -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
```

复制配置：

```bash
cp config.example.yaml config.yaml
```

如果用在线大模型 API：

```bash
export DEEPSEEK_API_KEY="你的key"
```

编辑 `config.yaml`：

```yaml
llm:
  base_url: "https://api.deepseek.com/v1"
  api_key: "${DEEPSEEK_API_KEY}"
  model: "deepseek-chat"
```

## 5. 验证命令

先确认工程可运行：

```bash
python evaluate_pasa.py --config config.smoke.json --input data/sample_eval.jsonl --output_dir runs/smoke_eval --no_llm --fallback_models --top_k 20
```

再确认在线 API 可用：

```bash
python run_agent.py --config config.online.json --query "intelligent vehicle cybersecurity risk assessment" --output runs/online_test.json --no_llm --fallback_models --top_k 10
```

最后跑正式配置：

```bash
python evaluate_pasa.py --config config.yaml --input data/RealScholarQuery/test.jsonl --output_dir runs/pasa_realscholar --limit 50
```

确认没问题后去掉 `--limit 50`。

## 6. 省钱建议

- 先租 RTX 4090 24GB 跑通流程，不要一开始上 A100。
- 如果模型加载稳定、显存不爆，就继续用 4090。
- 如果要本地跑 7B LLM 或批量评测很慢，再换 L20 48GB 或 A100。
- 能用在线 LLM API 时，不要在服务器上本地部署大模型，服务器只跑小模型和检索评测即可。

## 7. 常见问题

显存不足：

- 调小 `embedding_batch_size`。
- 调小 `reranker_batch_size`。
- 降低 `llm_verify_top_n`。
- 先加 `--fallback_models` 跑通流程。

内存不足：

- 降低 `max_candidates`。
- 降低 `per_query`。
- 分批运行测试集。

API 慢或失败：

- 配置 `SEMANTIC_SCHOLAR_API_KEY`。
- 设置 OpenAlex mailto。
- 减小 `max_api_calls_per_query`。
